## ✅ 한 줄씩 다시 정리한 이해 (확인용)

### 작업 1 — 주문자별 단가 (직거래 / 택배 따로)
- 주문자(사람)마다 단가가 달라야 하고, 관리자가 **지정**할 수 있어야 함.
- 같은 사람이라도 **직거래 가격 ≠ 택배 가격** → 사람마다 *직거래용 단가* / *택배용 단가* **2개를 각각** 지정 가능해야 함.
- ⚠️ 현재 `user_prices` 테이블은 `rep_price` **하나만** 있음 → 직거래/택배 구분 컬럼 추가 필요.

### 작업 2 — 가입 먼저 / 단가 지정 전 주문 = 틀린 가격 문제
- 새 사람의 **기본 베이스 가격은 사용자(관리자)가 직접 다르게 설정할 것** → 이건 내 작업 아님, 사용자가 함.
- **진짜 다듬을 부분:** 새 사람이 *가입을 먼저 하고*, 관리자가 단가를 지정하기 **전에** 주문을 넣으면 → 틀린(기본) 가격으로 주문이 들어가 버림.
- 이 **타이밍 문제**를 다듬어야 함. (예: 단가 미지정 사용자 주문 표시/보류, 나중에 단가 지정 시 소급 반영 등 — 구체안은 작업 시작할 때 논의)

### 작업 3 — 관리자 모드 UI 미니멀 정리
- 관리자 모드 아래쪽 탭들이 너무 난잡하고 뜻 모를 항목이 많음.
- 실제 쓰는 것 위주로 정리, **안 쓰는 건 삭제**, 미니멀하게.
- **관리자 관점이 아닌 "실제 사용자 관점"** 으로 재구성.
