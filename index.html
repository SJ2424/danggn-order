⚙️ 상품 카탈로그 + 단가

원가 = 본인이 공급사에 지불하는 매입가. 기본 납품가 = 일반 입력자한테 받는 가격. 마진 = (납품가 − 원가) × 수량.

👥 사용자별 특별 단가

설정 안 한 사용자는 위 기본 납품가 적용. 특별 가격 받는 사람만 등록.

🔑 사용자 비밀번호 재설정 / 푸시 알림

입력자가 비밀번호 잊었을 때 — Supabase 대시보드에서 직접 재설정합니다.

  1. 아래 [Supabase 사용자 관리] 버튼 → Authentication → Users
  2. 해당 사용자 이메일 행 우측의 ··· 점 3개 → Reset password
  3. 새 비밀번호 입력 (6자 이상)
  4. 입력자한테 새 비번 알려주기 (카톡 등)
↗ Supabase 사용자 관리 열기

📦 백업 / 유지보수 / 관리자 변경

💾 데이터 위치

  • 코드: GitHub (자동 버전관리, 영구) → danggn-order
  • 데이터(주문/사용자): Supabase Postgres (매일 자동 백업, 7일 보관)
  • 비밀 키: GitHub Secrets (암호화 저장)

📥 권장 월별 수동 백업

매월 말 [주문 관리·장부] → 도구바 [CSV 내보내기] → 본인 PC/구글드라이브에 보관.

🔑 관리자 이메일 변경

  1. Supabase 대시보드 → Authentication → Users → 본인 행 점 3개 → Send password recovery 또는 Change email
  2. 새 이메일로 인증 메일 옴 → 클릭
  3. 다음 로그인부터 새 이메일 사용
  4. 관리자 권한도 같이 옮기려면: SQL Editor에서
    update profiles set role='admin' where email='새이메일'; update profiles set role='input' where email='lidekorea@gmail.com';

📝 표시 이름 변경 (본인 또는 입력자)

Supabase → Table Editor → profiles → 해당 행 → display_name 셀 더블클릭 → 새 이름 입력.

🛠 코드 수정 (개발자/외주)

GitHub 저장소(SJ2424/danggn-order)에서 index.html 또는 bot/*.js 수정 → commit하면 Vercel 자동 재배포. 봇 변경은 GitHub Actions에서 자동 반영.

🛠 파싱 키워드 + 알람 테스트

영업일 12:30부터 5분마다 울립니다. 입력 마감 12:50 / 주문 마감 12:55.
